## 这是什么

这是一款**桌面端语音输入工具**的全套界面设计稿（视觉 + 交互稿）。
用户通过全局快捷键唤起小窗，说一段话 → AI 转文字 → 一键润色 / 翻译 → 复制到任意应用。

所有界面都是 HTML 文件，独立可打开，不依赖框架。每个文件对应软件中的一个真实界面。

## 视觉系统（必须严格遵守）

风格定位：**现代极简**（参考 Linear、Vercel、Notion 2024、Stripe docs）。
所有颜色、字体、圆角、阴影都在每个 HTML 的 `:root` 里，请直接复用，不要自行改色。

### 颜色 tokens（OKLch 格式，开发请用其转换后的等价值）

```css
:root {
  --bg:           oklch(99% 0.002 240);   /* 页面底色（近白，略冷） */
  --surface:      oklch(100% 0 0);        /* 卡片 / 浮窗表面，纯白 */
  --surface-2:    oklch(97% 0.003 250);   /* 次级表面（侧边栏底、列表行 hover） */
  --border:       oklch(92% 0.005 250);   /* 默认分隔线 / 边框 */
  --border-strong:oklch(86% 0.008 250);   /* 强分隔线 / 输入框边 */
  --muted:        oklch(54% 0.012 250);   /* 次级文字 / 说明 */
  --muted-2:      oklch(70% 0.008 250);   /* 三级文字 / placeholder */
  --fg:           oklch(18% 0.012 250);   /* 主要文字 */
  --fg-soft:      oklch(30% 0.012 250);   /* 次要正文 */
  --accent:       oklch(58% 0.18 255);    /* 主强调色：蓝紫 */
  --accent-soft:  oklch(96% 0.02 255);    /* 强调色的浅底（用于 badge、选中态） */
  --success:      oklch(60% 0.14 160);    /* 成功 / 在线 */
  --warning:      oklch(75% 0.14 75);     /* 警告 */
}
```

OKLch 现代浏览器（Chrome 111+、Safari 16.4+、Electron 25+、Tauri webview）原生支持。如果工具链不支持，请用对应的 hex / rgb 等价值，**不要凭印象选色**，从 HTML 文件里取真实渲染值。

### 字体栈

```css
--font-display: -apple-system, BlinkMacSystemFont, 'SF Pro Display', system-ui, sans-serif;
--font-body:    -apple-system, BlinkMacSystemFont, 'SF Pro Text', system-ui, sans-serif;
--font-mono:    ui-monospace, 'SF Mono', 'JetBrains Mono', Menlo, monospace;
```

全部使用系统字体，**不要引入 Inter / Roboto / 思源黑体**。
标题用 `--font-display`，正文用 `--font-body`，数字 / 时间 / 快捷键 / 代码用 `--font-mono`。
数字一律加 `font-variant-numeric: tabular-nums`（等宽数字）。

### 圆角 / 阴影

```css
--radius-sm: 6px;    /* 小按钮、tag */
--radius:    10px;   /* 输入框、常规卡片 */
--radius-lg: 14px;   /* 浮窗、大卡片 */

--shadow-sm: 0 1px 2px rgba(15,23,42,.04), 0 1px 1px rgba(15,23,42,.03);
--shadow-md: 0 4px 6px -1px rgba(15,23,42,.06), 0 2px 4px -2px rgba(15,23,42,.04);
--shadow-lg: 0 20px 40px -12px rgba(15,23,42,.12), 0 8px 16px -4px rgba(15,23,42,.06);
```

阴影偏冷蓝灰、克制；不要加彩色 glow，不要厚阴影。

## 关键交互

下面列每个界面里设计稿已经体现、开发时务必照做的交互：

### 01 录音浮窗
- 全屏置顶、可拖动、点击外部不消失（除非松开快捷键）
- 显示实时音波动画 + 录音计时（mm:ss）
- 至少 4 个状态：待机 / 录音中 / 处理中（转文字）/ 错误
- 失焦不隐藏，由用户主动停止或快捷键控制

### 02 结果浮窗
- 三个 Tab：原文 / 润色 / 翻译（点击切换）
- 主操作按钮：复制、粘贴到光标位置（需要软件焦点切回原应用）
- 二级操作：重新润色、换风格、换翻译目标语
- 关闭后这次结果应进入历史记录

### 03 菜单栏图标弹出层
- macOS 用 `NSPopover` 风格，Windows 用类似贴边小窗
- 毛玻璃背景（macOS: `vibrancy: 'menu'`，Windows: Mica/Acrylic）
- 主开关、今日数据、最近 3-5 条转录、入口到主窗口

### 04 主窗口
- 三栏布局：侧边栏（约 240px）+ 列表（约 360px）+ 详情（自适应）
- 列表支持搜索、按时间分组（今天 / 昨天 / 本周 / 更早）
- 详情页支持复看、重新润色 / 翻译、删除、收藏
- 窗口最小尺寸建议 960×640

### 05–08 设置
- 左侧 Tab 切换不同设置分类
- 任何更改即时生效，不需要"保存"按钮
- 危险操作（注销、清空历史）需二次确认弹窗

### 09 引导
- 仅首次启动出现，可跳过
- 第 2 步需要真正能录入快捷键
- 第 3 步需要请求麦克风权限

## 需要工程师拿主意的事

设计稿里没决定的：

1. **技术栈**：建议 **Tauri**（更小、更省内存，原生快捷键好做）或 **Electron**（生态最熟）。如选 SwiftUI / 原生，请用界面里的颜色和布局完整重绘。
2. **全局快捷键库**：macOS / Windows / Linux 都需要支持，可用 `tauri-plugin-global-shortcut` 或 `electron-globalShortcut`。
3. **语音识别 / AI 接入**：界面已留好 GPT / Claude / Llama 等选项。请决定本地（Whisper.cpp）还是云端（OpenAI / Azure / Deepgram），并实现切换。
4. **历史数据存储**：建议 SQLite 本地存储；如需多端同步再加云端。
5. **悬浮窗实现**：macOS 用 `NSPanel` + `.floating` level；Windows 用 `WS_EX_TOPMOST`；Tauri 配 `alwaysOnTop: true` + `skipTaskbar: true`。
6. **毛玻璃**：macOS 直接用 vibrancy；Windows 11 用 Mica；Linux 退回纯色。

## 不要做的事

- 不要改色板（不要把蓝紫换成蓝、绿、橙……）
- 不要引入新字体
- 不要把圆角加大或抹平
- 不要加渐变背景、彩色阴影、霓虹光效
- 不要在按钮、卡片、标题前加 emoji 图标（设计里没有）
- 不要把功能图标换成 emoji
- 不要把任何 placeholder 数据当真实数据展示（"3,247 次转录"等是示例文案）
